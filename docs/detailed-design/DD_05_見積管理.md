#### 1.1.3 カスタムメソッド

```java
/**
 * 論理削除された明細を除外して返却する。
 * Thymeleafテンプレートからの呼び出し時に使用。
 * del_flg='0'（有効）の明細のみをフィルタリングして返す。
 */
public List<TEstimateDetail> getTEstimateDetailList() {
    if (this.tEstimateDetailList.isEmpty()) {
        return this.tEstimateDetailList;
    }
    return this.tEstimateDetailList.stream()
        .filter(detail -> Flag.NO.getValue().equals(detail.getDelFlg()))
        .collect(Collectors.toList());
}

/**
 * 見積ステータスの表示名を返却する。
 * EstimateStatus列挙型のcode→valueの変換を行う。
 * パース不可の場合は「見積明細が正しく保存されていません」を返す。
 */
public String getEstimateStatusDisplay() {
    EstimateStatus estimateStatus1 = EstimateStatus.purse(this.estimateStatus);
    if (estimateStatus1 == null) {
        return "見積明細が正しく保存されていません";
    }
    return estimateStatus1.getValue();
}
```

- `hashCode`: `getClass().hashCode()` による固定値（Hibernateベストプラクティス準拠）

#### 2.1.3 初期表示処理シーケンス

```
GET /estimateList
    |
    v
EstimateListController.estimateList(Model)
    |
    +-- init(Model)
    |       |
    |       +-- commonService.getShopMap() --> ショップマップをModelに設定
    |       +-- EstimateStatus.toMap() --> 見積ステータスマップをModelに設定
    |       +-- new EstimateListForm()
    |       |       |
    |       |       +-- estimateStatus初期値: ["00"(作成), "20"(修正)]
    |       |
    |       +-- Model.addAttribute("estimateListForm", form)
    |
    v
return "estimate/estimate_list"
```

#### 2.1.4 検索処理シーケンス

```
POST /estimateList
    |
    v
findEstimateList(EstimateListForm, BindingResult, Model)
    |
    +-- init(Model)  // マスタデータ再設定
    |
    +-- partnerNo != null の場合
    |       +-- commonService.getMCompanyByPartnerNo() --> 得意先名を設定
    |
    +-- bindingResult.hasErrors() の場合 --> エラー表示（検索結果なし）
    |
    +-- tEstimateService.find(
    |       shopNo,
    |       estimateNo,
    |       partnerNo,
    |       goodsName,
    |       goodsCode,
    |       estimateStatus[],
    |       estimateDateFrom,    // uuuu/MM/dd厳密モードでパース
    |       estimateDateTo,
    |       priceChangeDateFrom,
    |       priceChangeDateTo,
    |       profitRate,
    |       Flag.NO              // del_flg='0'のみ
    |   )
    |
    +-- 検索結果を重複排除 + 見積日降順ソート
    |       estimateList.stream()
    |           .distinct()
    |           .sorted(Comparator.comparing(TEstimate::getEstimateDate).reversed())
    |           .collect(Collectors.toList())
    |
    +-- Model.addAttribute("estimateList", estimateList)
    |
    v
return "estimate/estimate_list"
```

**重要な実装ポイント**:
- 日付パースは `DateTimeFormatter.ofPattern("uuuu/MM/dd").withResolverStyle(ResolverStyle.STRICT)` を使用
- 検索結果のソートはJava側で実施（DB側のORDER BYは使用していない）
- `distinct()` は `TEstimate.equals()` による `estimateNo` の等値比較で重複排除

#### 2.1.5 Ajax ステータス更新処理シーケンス

```
POST /updateEstimateStatusForAjax
    requestParam: estimateNo(Integer), estimateStatus(String)
    |
    v
updateEstimateStatusForAjax(estimateNo, estimateStatus)
    |
    +-- tEstimateService.getByEstimateNo(estimateNo)
    |       失敗時 --> return 0
    |       null時 --> return 0
    |
    +-- tEstimate.setEstimateStatus(estimateStatus)
    |
    +-- tEstimateService.update(tEstimate)
    |       失敗時 --> return 0
    |
    v
return 1  // 成功
```

**戻り値**: `@ResponseBody` による JSON整数値（`1`=成功, `0`=失敗）

## 6. 利益・粗利率計算ロジック

### 6.1 計算式

**実装箇所**: `EstimateUtil.fillEstimateGoods()`

| 項目 | 計算式 | 端数処理 |
|------|--------|---------|
| 粗利（profit） | `goodsPrice - purchasePrice` | なし |
| 粗利率（profitRate） | `(1 - purchasePrice / goodsPrice) * 100` | 除算時: scale=3, `RoundingMode.HALF_UP` |
| ケース粗利（caseProfit） | `profit * containNum` | なし |

### 6.2 実装コード

```java
if (estimateDetailForm.getGoodsPrice() != null) {
    // 粗利 = 見積単価 - 原価
    BigDecimal profit = estimateDetailForm.getGoodsPrice()
        .subtract(vEstimateGoods.getPurchasePrice());

    // 粗利率(%) = (1 - 原価 / 見積単価) * 100
    //   除算のスケール: 小数点以下3桁、四捨五入
    BigDecimal profitRate = BigDecimal.ONE
        .subtract(vEstimateGoods.getPurchasePrice()
            .divide(estimateDetailForm.getGoodsPrice(), 3, RoundingMode.HALF_UP))
        .multiply(new BigDecimal(100));

    // ケース粗利 = 粗利 * 入数
    BigDecimal caseProfit = profit.multiply(caseContainNum);

    estimateDetailForm.setProfit(profit);
    estimateDetailForm.setProfitRate(profitRate);
    estimateDetailForm.setCaseProfit(caseProfit);
}
```

### 6.3 計算前提条件

- `goodsPrice`（見積単価）が `null` でない場合のみ計算を実行
- `goodsPrice` が `0` の場合、`ArithmeticException`（ゼロ除算）が発生する可能性がある点に注意

### 6.4 税込変換計算式

**実装箇所**: `EstimateUtil.settingEstimateDetailList()`

```
税率選択:
  applyReducedTaxRate == true  --> reducedTaxRate（軽減税率: 8%等）
  applyReducedTaxRate == false --> taxRate（標準税率: 10%等）

税率掛け率 = 1 + taxRate / 100
  (除算スケール: 3桁, HALF_UP)

税込単価 = goodsPrice * 税率掛け率
  (端数処理: scale=0, RoundingMode.DOWN = 切り捨て)
```

### 6.5 入数選択ロジック

```
1. changeContainNum が設定されている場合 --> changeContainNum を使用
2. caseContainNum が設定されている場合   --> caseContainNum を使用
3. それ以外                              --> BigDecimal.ONE（1）を使用
```

入数変更がある場合（`changeContainNum != null`）かつ `detailNote` に「入数変更」が未含の場合:
```
detailNote = "{既存のdetailNote} 入数変更{旧入数}→{新入数}"
```

### 6.6 原価改訂予定情報の表示

`changePlanDate` が設定されている場合:
```
purchasePriceChangePlan = "{changePlanDate}より{beforePrice}→{afterPrice}"
例: "2024-04-01より100.00→120.00"
```

## 7. 見積ステータス管理

### 7.1 EstimateStatus 列挙型定義

**パッケージ**: `jp.co.oda32.constant`

| コード | 定数名 | 表示名 | 説明 |
|--------|--------|--------|------|

protected EstimateStatus getNotifiedStatus(String estimateStatusCode) {

}

## 8. 親子得意先間の見積同期設計

### 8.1 概要

`m_partner.parent_partner_no` により親子関係が定義される得意先間で、見積の登録・修正時に自動的に見積内容を同期する。同期処理は `EstimateCreateController` 内で見積登録（`/estimateCreate`）の処理後に実行される。

### 8.2 同期パターン判定ロジック

```
見積登録後:
    |
    +-- MPartner取得（登録した見積の得意先）
    |
    +-- parentPartnerNo != null の場合（子得意先の見積を登録した）
    |       |
    |       +-- beforeChangeEstimate != null（修正）
    |       |       --> updateParentEstimate()  // パターン1-A: 親見積を更新
    |       |
    |       +-- beforeChangeEstimate == null（新規）
    |               --> createParentEstimate()  // パターン1-B: 親見積を新規作成
    |
    +-- parentPartnerNo == null の場合
            |
            +-- findChildrenPartner(partnerNo) --> 子得意先リスト
            +-- 子得意先リストが空でない場合（親得意先の見積を登録した）
            |       |
            |       +-- beforeChangeEstimate != null（修正）
            |       |       --> updateChildrenEstimate()  // パターン2-A: 子見積を更新
            |       |
            |       +-- beforeChangeEstimate == null（新規）
            |               --> createChildrenEstimate()  // パターン2-B: 子見積を新規作成
            |
            +-- 子得意先リストが空の場合
                    --> 同期なし
```

### 8.3 パターン1-A: 子得意先見積修正時の親見積更新

**メソッド**: `updateParentEstimate(childrenEstimate, beforePriceChangeDate, parentPartnerNo)`

```
1. 親得意先の提出前見積を検索
    条件: shopNo + parentPartnerNo + beforePriceChangeDate
    ステータス: getBeforeNotifiedStatusCodeList() = ["00", "20"]

2. FOR EACH 親見積:
    a. 明細を物理削除
       tEstimateDetailService.delete(parentEstimate.getEstimateNo())

    b. 明細リストをクリア
       parentEstimate.setTEstimateDetailList(new ArrayList<>())

    c. ヘッダの更新
       priceChangeDate = 子見積のpriceChangeDate
       estimateDate = 子見積のestimateDate

    d. 子見積の明細を親見積にコピー
       FOR EACH 子見積明細:
           BeanUtils.copyProperties(childrenDetail, parentDetail)
           parentDetail.setEstimateNo(parentEstimate.getEstimateNo())

    e. 見積明細番号の再編成 (rebuildTEstimateDetailNo)

    f. 明細を保存 + ヘッダを更新

3. 全ての兄弟（子得意先）の見積にも反映
   findChildrenPartner(parentPartnerNo) --> 子得意先リスト
   FOR EACH 親見積:
       updateChildrenEstimate(parentEstimate, beforePriceChangeDate, childrenPartnerList)
```

### 8.4 パターン1-B: 子得意先見積新規作成時の親見積新規作成

**メソッド**: `createParentEstimate(childrenEstimate, parentPartnerNo)`

```
1. 子見積をコピーして親見積を作成
    BeanUtils.copyProperties(childrenEstimate, parentEstimate)
    parentEstimate.setEstimateNo(null)     // 新規採番
    parentEstimate.setPartnerNo(parentPartnerNo)

2. tEstimateService.insert(parentEstimate)

3. 子見積の明細をコピーして親明細を作成
    FOR EACH 子見積明細:
        BeanUtils.copyProperties(childrenDetail, parentDetail)
        parentDetail.setEstimateNo(parentEstimate.getEstimateNo())

4. 見積明細番号の再編成 (rebuildTEstimateDetailNo)

5. 明細を保存

6. 全ての子得意先にも反映
    updateChildrenEstimate(parentEstimate, parentEstimate.getPriceChangeDate(), childrenPartnerList)
```

### 8.5 パターン2-A: 親得意先見積修正時の子見積更新

**メソッド**: `updateChildrenEstimate(parentEstimate, beforePriceChangeDate, childrenPartnerList)`

```
FOR EACH 子得意先:
    1. 子得意先の提出前見積を検索
       条件: shopNo + childrenPartner.partnerNo + beforePriceChangeDate
       ステータス: ["00", "20"]

    2. FOR EACH 子見積:
        a. 明細を物理削除
        b. 明細リストをクリア
        c. ヘッダの更新
           priceChangeDate = 親見積のpriceChangeDate
           estimateDate = 親見積のestimateDate
           estimateStatus = OTHER_PARTNER_NOTIFIED("40")  ★ 他同グループ提出済に変更
        d. 親見積の明細をコピーして子明細を作成
        e. 見積明細番号の再編成
        f. 明細を保存 + ヘッダを更新
```

### 8.6 パターン2-B: 親得意先見積新規作成時の子見積新規作成

**メソッド**: `createChildrenEstimate(parentEstimate, childrenPartnerList)`

```
FOR EACH 子得意先:
    1. 親見積をコピーして子見積を作成
       estimateNo = null（新規採番）
       partnerNo = childrenPartner.getPartnerNo()
       estimateStatus = OTHER_PARTNER_NOTIFIED("40")  ★ 他同グループ提出済

    2. tEstimateService.insert(childrenEstimate)

    3. 親見積の明細をコピーして子明細を作成
    4. 見積明細番号の再編成
    5. 明細を保存
```

### 8.7 見積明細番号の再編成ロジック

**メソッド**: `rebuildTEstimateDetailNo(List<TEstimateDetail>)`

```java
private void rebuildTEstimateDetailNo(List<TEstimateDetail> tEstimateDetailList) {
    // estimate_noごとにグルーピング
    Map<Integer, List<TEstimateDetail>> map = tEstimateDetailList.stream()
        .collect(Collectors.groupingBy(TEstimateDetail::getEstimateNo));

    for (Map.Entry<Integer, List<TEstimateDetail>> entry : map.entrySet()) {
        List<TEstimateDetail> details = entry.getValue();
        // goods_codeの昇順でソート
        details.sort(Comparator.comparing(TEstimateDetail::getGoodsCode));
        // 1から連番を振り直す
        int i = 1;
        for (TEstimateDetail detail : details) {
            detail.setEstimateDetailNo(i);
            i++;
        }
    }
}
```

### 8.8 同期対象・除外条件まとめ

| 条件 | 値 |
|------|-----|
| 同期対象の見積ステータス | 提出前: `["00"(作成), "20"(修正)]` |
| 同期対象の削除フラグ | `del_flg = '0'`（有効のみ） |
| 価格変更日の一致条件 | 修正時: `beforePriceChangeDate` で検索。新規時: 現在の `priceChangeDate` で検索 |
| 子見積の同期後ステータス | `OTHER_PARTNER_NOTIFIED("40")` に変更 |
