### 6.3 GoodsFileReader 処理フロー

**パッケージ**: `jp.co.oda32.batch.goods`
**アノテーション**: `@Component`, `@RequiredArgsConstructor`, `@Log4j2`
**インターフェース**: `ItemReader<GoodsFile>`

#### 処理シーケンス（`@BeforeStep`）

```
1. MShopLinkedFileService#findAll()
   |
2. smileGoodsImportFileName が空でないレコードに絞込
   |
3. 各ファイルパスの存在チェック
   |  - 存在しない場合: log.warn でスキップ
   |
4. MultiResourceItemReader<GoodsFile> を初期化
   |  - setStrict(false): ファイル0件でもエラーにしない
   |  - setResources(): FileSystemResource配列を設定
   |
5. FlatFileItemReader<GoodsFile> をデリゲートに設定
   |  - setEncoding("Unicode")
   |  - setLineMapper: DelimitedLineTokenizer + BeanWrapperFieldSetMapper
   |    - setDistanceLimit(0): 日本語フィールド名の曖昧一致を無効化
   |
6. reader.open(jobContext)
```

#### ファイル取得元

`m_shop_linked_file.smile_goods_import_file_name` にショップ別のCSVファイルパスが設定されている。

### 6.4 GoodsFileProcessor 処理フロー

**パッケージ**: `jp.co.oda32.batch.goods`
**アノテーション**: `@Component`
**インターフェース**: `ItemProcessor<GoodsFile, GoodsFile>`

#### フィルタリング・加工ルール

| # | 条件 | 処理 | 戻り値 |
|---|---|---|---|
| 1 | `商品名`がnull/空白 | スキップ | `null` |
| 2 | `商品名`に「休止」を含む | スキップ | `null` |
| 3 | `商品コード` == `"99999999"` | `商品名`をUTF-8バイト列でMD5ハッシュ化し`商品コード`に設定 | 加工後item |
| 4 | `入数` < 1 | `BigDecimal.ONE`を設定 | 加工後item |
| 5 | `ＪＡＮ`の桁数が8桁/13桁以外 | `null`を設定 | 加工後item |
| 6 | `標準売上単価` <= 0 | `null`を設定 | 加工後item |
| 7 | `標準仕入単価` <= 0 | `null`を設定 | 加工後item |

**MD5ハッシュ化処理**:
```java
DigestUtils.md5DigestAsHex(item.get商品名().getBytes())
```

### 6.5 GoodsFileWriter 処理フロー

**パッケージ**: `jp.co.oda32.batch.goods`
**アノテーション**: `@Component`, `@StepScope`, `@Log4j2`
**インターフェース**: `ItemWriter<GoodsFile>`

#### `write()`メソッドのシーケンス

```
for each GoodsFile item:
  1. ログ出力（keyword, maker_code, メーカー名, 商品コード, 商品名）
  2. GoodsFileWriterFactory#getGoodsFileProcessorLogic(item)
     -> IGoodsFileWriterLogic を取得
  3. IGoodsFileWriterLogic#register()
     -> 新規 or 既存の処理を実行
```

### 6.6 GoodsFileWriterFactory 分岐ロジック

**パッケージ**: `jp.co.oda32.batch.goods.logic`
**アノテーション**: `@Service`, `@Scope("step")`
**ジョブパラメータ**: `shopNo`（`@Value("#{jobParameters['shopNo']}")`）

#### 分岐判定フロー

```
getGoodsFileProcessorLogic(item):
  |
  +-- getExistWSalesGoods(商品コード)
  |     |
  |     +-- WSalesGoodsService#getByShopNoAndGoodsCode(shopNo, goodsCode)
  |     |     -> w_sales_goods に存在する場合: 返却（isWork=true）
  |     |
  |     +-- MSalesGoodsService#getByShopNoAndGoodsCode(shopNo, goodsCode)
  |           -> m_sales_goods に存在する場合: 返却（isWork=false）
  |           -> どちらにも存在しない場合: null
  |
  +-- existSalesGoods != null の場合:
  |     ExistGoodsWriterLogic#setGoodsData(shopNo, item, existSalesGoods)
  |     return ExistGoodsWriterLogic
  |
  +-- existSalesGoods == null の場合:
        NewGoodsWriterLogic#setGoodsData(shopNo, item, null)
        return NewGoodsWriterLogic
```

#### 検索優先順位

| 優先度 | テーブル | 条件 | ISalesGoods#getIsWork() |
|---|---|---|---|
| 1 | `w_sales_goods` | `shop_no` + `goods_code` | `true` |
| 2 | `m_sales_goods` | `shop_no` + `goods_code` | `false` |
| 3 | 該当なし | - | 新規商品として処理 |

### 6.8 AbstractGoodsFileWriterLogic（基底クラス）

**パッケージ**: `jp.co.oda32.batch.goods.logic`
**アノテーション**: `@Service`, `@RequiredArgsConstructor`

#### 保持フィールド

| フィールド | 型 | 説明 |
|---|---|---|
| `goodsService` | `MGoodsService` | 商品マスタサービス |
| `goodsUnitService` | `MGoodsUnitService` | 商品単位サービス |
| `supplierService` | `MSupplierService` | 仕入先サービス |
| `makerService` | `MMakerService` | メーカーサービス |

| `shopNo` | `Integer` | 処理対象ショップ番号 |
| `goodsFile` | `GoodsFile` | 入力データ |
| `iSalesGoods` | `ISalesGoods` | 既存販売商品（既存商品の場合） |
| `mGoods` | `MGoods` | 商品マスタEntity |
| `UNIT` | `UnitType` | 固定値: `UnitType.CASE` |
| `existMakerMap` | `Map<String, MMaker>` | メーカー検索キャッシュ |

#### `setGoodsData()` メソッド

```java
public void setGoodsData(Integer shopNo, GoodsFile goodsFile, ISalesGoods iSalesGoods)
```

- `shopNo`, `goodsFile` を設定
- `iSalesGoods != null` の場合: `this.iSalesGoods` と `this.mGoods`（`iSalesGoods.getMGoods()`）を設定

#### `saveMaker()` 共通実装

```
メーカー名が空 -> return
  |
existMakerMap にキャッシュあり -> 既存メーカー取得
  |  なければ MMakerService#findByMakerName で検索
  |
存在する場合:
  - del_flg == '1' なら '0' に戻して update
存在しない場合:
  - MMaker を新規 insert し existMakerMap にキャッシュ
```

#### `saveMGoodsUnit()` 共通実装（新規商品用）

```
1. MGoodsUnit(unit=CASE, containNum=入数, goodsNo=商品番号) を insert
2. MGoodsUnit(unit=PIECE, containNum=1, parentUnitNo=上記のunitNo) を insert
```

ケース単位と個単位の親子関係を同時に作成する。

### 6.9 NewGoodsWriterLogic（新規商品処理）

**パッケージ**: `jp.co.oda32.batch.goods.logic`
**アノテーション**: `@Service`
**継承**: `AbstractGoodsFileWriterLogic`

#### `register()` 呼び出し順序

| 順序 | メソッド | 対象テーブル | 処理 |
|---|---|---|---|
| 1 | `saveMaker()` | `m_maker` | メーカー登録/復元 |
| 2 | `saveMGoods()` | `m_goods` | 商品マスタINSERT |
| 3 | `saveMGoodsUnit()` | `m_goods_unit` | CASE + PIECE単位INSERT |
| 4 | `saveSalesGoods()` | `w_sales_goods` | 販売商品WORK INSERT |

#### `saveMGoods()` 詳細

```
1. 税区分判定:
   - 非課税区分 == 1 -> TaxCategory.EXEMPT
   - それ以外 -> TaxCategory.purse(新税分類)
     （0=通常, 1=軽減税率）

2. MGoods.builder()
   - goodsName = 商品名
   - keyword = 商品名索引
   - janCode = ＪＡＮ
   - taxCategory = taxCategory.getCode()
   - caseContainNum = 入数
   - smileGoodsName = 商品名

3. メーカー名が空でなければ:
   - existMakerMap から makerNo を取得して設定

4. MGoodsService#insert(goods) -> mGoods に保持
```

#### `saveSalesGoods()` 詳細

```
WSalesGoods.builder()
  - goodsCode = 商品コード
  - goodsNo = mGoods.getGoodsNo()
  - keyword = mGoods.getKeyword()
  - shopNo = shopNo
  - goodsName = mGoods.getGoodsName()
  - purchasePrice = 標準仕入単価
  - goodsPrice = 標準売上単価
-> WSalesGoodsService#insert()
```

### 6.10 ExistGoodsWriterLogic（既存商品処理）

**パッケージ**: `jp.co.oda32.batch.goods.logic`
**アノテーション**: `@Service`, `@Log4j2`
**継承**: `AbstractGoodsFileWriterLogic`

#### `register()` 呼び出し順序

| 順序 | メソッド | 対象テーブル | 処理 |
|---|---|---|---|
| 1 | `saveMaker()` | `m_maker` | メーカー登録/復元 |
| 2 | `saveMGoods()` | `m_goods` | 差分UPDATE |
| 3 | `saveMGoodsUnit()` | `m_goods_unit` | 差分UPDATE/INSERT |
| 4 | `saveSalesGoods()` | `w_sales_goods` | 差分UPDATE/INSERT |

#### `saveMGoods()` 差分チェック・更新

| # | チェック項目 | 更新条件 | ログレベル |
|---|---|---|---|
| 1 | キーワード | 既存が空で入力ありの場合: 新規設定 / 既存に含まれない索引語: カンマ区切りで追加 | - |
| 2 | 商品名 | 不一致の場合 | `warn` |
| 3 | SMILE商品名 | 既存が空 or 不一致の場合 | `warn` |
| 4 | JANコード | 入力非null かつ不一致の場合 | `warn` |
| 5 | メーカー番号 | メーカー名非空 かつ existMakerMap に存在 かつ（既存null or 不一致）の場合 | `warn` |
| 6 | ケース入数 | 不一致の場合 | `warn` |
| 7 | 税区分 | 既存null or 不一致の場合 | `info` |

`isUpdate == true` の場合のみ `MGoodsService#update(mGoods)` を実行。

#### `saveSalesGoods()` 差分チェック・更新

```
1. BeanUtils.copyProperties(iSalesGoods, wSalesGoods) で既存データをコピー

2. 仕入単価の差分チェック:
   - 不一致 -> wSalesGoods.setPurchasePrice(新値), warn log

3. 売単価の差分チェック:
   - 不一致 -> wSalesGoods.setGoodsPrice(新値), warn log

4. 商品名の差分チェック:
   - 不一致 -> wSalesGoods.setGoodsName(新値), warn log

5. iSalesGoods.getIsWork() による分岐:
   - false (m_sales_goodsのみ存在): WSalesGoodsService#insert(wSalesGoods)
   - true (w_sales_goods存在): WSalesGoodsService#update(wSalesGoods)
```

#### `saveMGoodsUnit()` 差分チェック・更新

```
1. MGoodsUnitService#findByGoodsNo(goodsNo) で既存単位一覧取得
2. unit == CASE のレコードを検索
   - 存在しない場合: super.saveMGoodsUnit() で新規登録
   - 存在する場合:
     - 入数が同じ -> スキップ
     - 入数が異なる -> MGoodsUnitService#update(), warn log
```

### 6.11 SalesGoodsCreateService（仕入/注文時の商品自動登録）

**パッケージ**: `jp.co.oda32.batch.goods.logic`
**アノテーション**: `@Service`, `@Log4j2`, `@RequiredArgsConstructor`

仕入や注文登録時に未登録商品を自動作成する補助サービス。

#### `goodsProcess(ISmileGoodsFile)` メソッド

```
1. 商品コードが空 -> log.warn, return null
2. 税区分判定（課税区分が非課税なら EXEMPT、それ以外は消費税分類で判定）
3. MGoods を insert (goodsName, smileGoodsName, taxCategory, caseContainNum)
4. WSalesGoods を insert (goodsNo, goodsCode, goodsName, shopNo, purchasePrice)
5. return wSalesGoods
```
