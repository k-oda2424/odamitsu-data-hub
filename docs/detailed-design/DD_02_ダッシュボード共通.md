#### 処理フロー全体図

```
[8] テンプレート名"dashboard"を返却
```

#### STEP 1: ログインユーザー情報取得

```java
// SecurityContextHolderからLoginUserを取得
MLoginUser loginUser = LoginUserUtil.getLoginUserInfo().getUser();
// LoginUserUtil.getLoginUserInfo()は内部で以下を実行:
//   SecurityContextHolder.getContext().getAuthentication().getPrincipal()
//   → LoginUser にキャスト → LoginUser.getUser() で MLoginUser を返却
```

#### STEP 2: 会社種別判定・ショップ番号決定

```java
Integer loginUserShopNo = null;  // admin の場合は null のまま（全ショップ対象）
CompanyType ut = CompanyType.purse(loginUser.getCompanyType());
// CompanyType.purse() は文字列 → Enum変換
//   "admin"   → CompanyType.ADMIN
//   "shop"    → CompanyType.SHOP
//   "partner" → CompanyType.PARTNER
//   該当なし  → null → Exception

switch (ut) {
    case SHOP:
        loginUserShopNo = loginUser.getCompany().getShopNo();
        break;
    case PARTNER:
        loginUserShopNo = loginUser.getCompany().getPartner().getShopNo();
        break;
    // case ADMIN: loginUserShopNo は null のまま → 全ショップ検索
}
```

| 会社種別 | loginUserShopNo | データスコープ |
|---------|----------------|--------------|
| ADMIN | `null` | 全ショップの売上データを取得 |
| SHOP | `company.shopNo` | 自ショップのみ |
| PARTNER | `company.partner.shopNo` | 所属ショップのみ |

#### STEP 3: 売上データ取得（2年分）

```java
LocalDate now = LocalDate.now().plusMonths(1);  // 現在月 + 1（当月分を含むため翌月を上限とする）
LocalDate from = now.minusYears(2);             // 2年前

// Specificationを使った動的クエリ
List<VSalesMonthlySummary> salesList = vSalesMonthlySummaryService.find(
    loginUserShopNo,  // null の場合は shopNo 条件なし（全ショップ）
    null,             // month: 個別月指定なし
    from,             // dateFrom: 2年前
    now               // dateTo: 現在月+1
);
```

**呼び出し先のSpecification構成**:

```
WHERE
  (shopNo = :loginUserShopNo OR :loginUserShopNo IS NULL)  -- shopNoContains
  AND month BETWEEN :from('uuuu-MM') AND :now('uuuu-MM')   -- monthRangeContains
```

#### STEP 4: 複数ショップ合算処理

```java
// Stream API の Collectors.toMap で month をキーに合算
Map<String, VSalesMonthlySummary> salesMap = salesList.stream()
    .collect(Collectors.toMap(
        VSalesMonthlySummary::getMonth,      // キー: "2025-07" 等
        salesSummary -> salesSummary,          // 値: そのまま
        (sales1, sales2) -> {                 // マージ関数: 同一月データを合算
            sales1.setSalesTotal(
                sales1.getSalesTotal().add(sales2.getSalesTotal())
            );
            return sales1;
        }
    ));
List<VSalesMonthlySummary> salesSummaryList = new ArrayList<>(salesMap.values());
```

**合算ロジックの詳細**:
- adminユーザーの場合、同一月に複数ショップのレコードが存在する
- `Collectors.toMap` のマージ関数で `salesTotal` を `BigDecimal.add()` で合算
- 合算後は月ごとに1レコードに集約される

#### STEP 5: 会計期（7月始まり）算出ロジック

```
疑似コード:
  now = LocalDate.now().plusMonths(1)
  formatter = "uuuu-MM" (ResolverStyle.STRICT)

  IF now.月 < 7 THEN
    // 1月〜6月の場合: 今期は前年7月開始
    nowSeasonStart  = (now.年 - 1) + "-07"    // 例: now=2026-03 → "2025-07"
    lastSeasonStart = (now.年 - 2) + "-07"    // 例: now=2026-03 → "2024-07"
  ELSE
    // 7月〜12月の場合: 今期は当年7月開始
    nowSeasonStart  = now.年 + "-07"           // 例: now=2025-09 → "2025-07"
    lastSeasonStart = (now.年 - 1) + "-07"    // 例: now=2025-09 → "2024-07"
  END IF
```

**具体例**（現在日が2026年2月の場合）:

| 変数 | 計算過程 | 値 |
|------|---------|-----|
| `now` | `2026-02-23 + 1ヶ月` | `2026-03-23` |
| `now.getMonthValue()` | 3 | `3 < 7` → IF分岐 |
| `nowSeasonStart` | `(2026-1)年7月` | `"2025-07"` |
| `lastSeasonStart` | `(2026-2)年7月` | `"2024-07"` |

#### STEP 6: チャートデータ構築

```java
// 月ソート（"2024-07", "2024-08", ... の昇順）
salesSummaryList.sort(Comparator.comparing(VSalesMonthlySummary::getMonth));

// 横軸ラベル（会計期順で固定）
String[] labels = {"7月", "8月", "9月", "10月", "11月", "12月",
                   "1月", "2月", "3月", "4月", "5月", "6月"};

// 今期データ: nowSeasonStart 以降のデータを抽出
BigDecimal[] thisYearData = salesSummaryList.stream()
    .filter(s -> nowSeasonStart.compareTo(s.getMonth()) <= 0)
    .map(VSalesMonthlySummary::getSalesTotal)
    .toArray(BigDecimal[]::new);

// 前期データ: lastSeasonStart 以降 かつ nowSeasonStart 未満
BigDecimal[] lastYearData = salesSummaryList.stream()
    .filter(s -> nowSeasonStart.compareTo(s.getMonth()) > 0)
    .filter(s -> lastSeasonStart.compareTo(s.getMonth()) <= 0)
    .map(VSalesMonthlySummary::getSalesTotal)
    .toArray(BigDecimal[]::new);
```

**フィルタ条件の図解**:

```
時系列: ----[lastSeasonStart]---前期---[nowSeasonStart]---今期---[now]-->

lastYearData: lastSeasonStart <= month < nowSeasonStart
thisYearData: nowSeasonStart <= month
```

#### STEP 7: Model属性設定

| Model属性名 | 型 | 値 | テンプレート側での用途 |
|------------|-----|-----|-------------------|
| `thisYearLabel` | `String` | `"今期"` | Chart.js datasets[0].label |
| `labels` | `String[]` | `{"7月", "8月", ..., "6月"}` | Chart.js data.labels |
| `thisYearData` | `BigDecimal[]` | 今期月別売上配列 | Chart.js datasets[0].data |
| `lastYearLabel` | `String` | `"前期"` | Chart.js datasets[1].label |
| `lastYearData` | `BigDecimal[]` | 前期月別売上配列 | Chart.js datasets[1].data |

#### STEP 8: 戻り値

```java
return "dashboard";  // → templates/dashboard.html を解決
```

## 3. BatchStatusAPI 処理設計

### 3.1 クラス定義

| 項目 | 値 |
|------|-----|

| アノテーション | `@RestController` |

### 3.2 getJobStatus() メソッド

```java
@ResponseBody
@RequestMapping("/job/status/{jobId}")
public BatchStatus getJobStatus(@PathVariable String jobId)
```

#### 処理フロー

```
[1] パスパラメータ jobId を受け取る（String型）
    |
[2] Long型にパース
    |--- NumberFormatException → BatchStatus.UNKNOWN を返却
    |
[3] jobExplorer.getJobExecution(jobIdL) で JobExecution を取得
    |--- null → BatchStatus.UNKNOWN を返却
    |
[4] jobExecution.getStatus() を返却
```

#### 処理フロー疑似コード

```
FUNCTION getJobStatus(jobId: String) -> BatchStatus
  TRY
    jobIdL = Long.parseLong(jobId)
  CATCH NumberFormatException
    RETURN BatchStatus.UNKNOWN
  END TRY

  jobExecution = jobExplorer.getJobExecution(jobIdL)

  IF jobExecution == null THEN
    RETURN BatchStatus.UNKNOWN
  END IF

  RETURN jobExecution.getStatus()
END FUNCTION
```

#### レスポンス値一覧

| BatchStatus | 意味 | クライアント側処理 |
|-------------|------|-----------------|
| `COMPLETED` | ジョブ正常完了 | ステータステキストを「完了」に変更 |
| `FAILED` | ジョブ実行失敗 | ステータステキストを「実行失敗」に変更 |
| `STARTED` | ジョブ実行中 | 5秒後に再ポーリング |
| `STARTING` | ジョブ起動中 | 5秒後に再ポーリング |
| `STOPPING` | ジョブ停止中 | 5秒後に再ポーリング |
| `STOPPED` | ジョブ停止 | 5秒後に再ポーリング |
| `ABANDONED` | ジョブ放棄 | 5秒後に再ポーリング |
| `UNKNOWN` | ジョブ不明 | 5秒後に再ポーリング |

#### JobExplorer のステータス取得フロー（内部動作）

```
BatchStatusAPI
  |
  +-- jobExplorer.getJobExecution(jobIdL)
        |
        +-- Spring Batch メタデータテーブル参照
              |
              +-- SELECT * FROM BATCH_JOB_EXECUTION WHERE JOB_EXECUTION_ID = ?
                    |
                    +-- JobExecution オブジェクト構築
                          |
                          +-- getStatus() → STATUS カラムの値を BatchStatus にマッピング
```

#### ポーリング設計

| 項目 | 値 |
|------|-----|
| ポーリング間隔 | 5,000ms（5秒） |
| ポーリング停止条件 | `COMPLETED` または `FAILED` |
| タイムアウト | 明示的タイムアウトなし（ジョブが終了するまで継続） |
| エラー時挙動 | Ajax error コールバックでエラーメッセージ表示 |

## 4. 売上チャートデータ生成設計

### 4.1 データフロー全体図

```
[PostgreSQL]                    [Java]                          [JavaScript]
v_sales_monthly_summary    DashboardController              dashboard.html (Chart.js)
(マテリアライズドビュー)
                           VSalesMonthlySummaryService
  shop_no | month | sales_total
  --------+-------+------------
    1     |2025-07| 1500000     →  find(shopNo, null, from, now)
    1     |2025-08| 2000000         |
    2     |2025-07|  500000         ↓
    2     |2025-08|  800000     Collectors.toMap (合算)
                                    |
                                    ↓
                                month  | salesTotal         → Thymeleaf変数式
                                -------+-----------            で展開
                                2025-07| 2000000   (1500000+500000)
                                2025-08| 2800000   (2000000+800000)
                                    |
                                    ↓
                                会計期フィルタ
                                    |
                                +---------+----------+
                                |                    |
                           thisYearData[]      lastYearData[]
                                |                    |
                                ↓                    ↓
                           Chart.js datasets[0]  datasets[1]
```

### 4.2 VSalesMonthlySummary エンティティ

```java
@Entity
@Table(name = "v_sales_monthly_summary")
@IdClass(VSalesMonthlySummaryPK.class)
public class VSalesMonthlySummary {
    @Id @Column(name = "shop_no")    protected Integer shopNo;
    @Id @Column(name = "month")      protected String month;      // "YYYY-MM" 形式
    @Column(name = "sales_total")    protected BigDecimal salesTotal;
}
```

**複合主キークラス**: `VSalesMonthlySummaryPK`
- `shopNo` (Integer) + `month` (String)

### 4.3 VSalesMonthlySummaryService

```java
@Service
@RequiredArgsConstructor
public class VSalesMonthlySummaryService {
    final VSalesMonthlySummaryRepository vSalesMonthlySummaryRepository;
    VSalesMonthlySummarySpecification vSalesMonthlySummarySpecification
        = new VSalesMonthlySummarySpecification();

    public List<VSalesMonthlySummary> find(
        Integer shopNo,       // null: 全ショップ
        LocalDate month,      // null: 個別月指定なし
        LocalDate dateFrom,   // 検索開始月
        LocalDate dateTo      // 検索終了月
    ) {
        return vSalesMonthlySummaryRepository.findAll(
            Specification
                .where(vSalesMonthlySummarySpecification.shopNoContains(shopNo))
                .and(vSalesMonthlySummarySpecification.monthContains(month))
                .and(vSalesMonthlySummarySpecification.monthRangeContains(dateFrom, dateTo))
        );
    }
}
```

### 4.4 Specification 構成

| Specificationメソッド | 引数 | 生成条件 |
|---------------------|------|---------|
| `shopNoContains(Integer)` | `null` → 条件なし | `WHERE shop_no = :shopNo` |
| `monthContains(LocalDate)` | `null` → 条件なし | `WHERE month = :month` |
| `monthRangeContains(LocalDate, LocalDate)` | 両方`null` → 条件なし | `WHERE month BETWEEN :from AND :to` |

**DateTimeFormatterの統一**: すべて `"uuuu-MM"` + `ResolverStyle.STRICT` を使用。

### 4.5 複数ショップの合算ロジック（Stream API）

```java
// 入力例（adminの場合）:
// [{shopNo:1, month:"2025-07", salesTotal:1500000},
//  {shopNo:2, month:"2025-07", salesTotal:500000},
//  {shopNo:1, month:"2025-08", salesTotal:2000000},
//  {shopNo:2, month:"2025-08", salesTotal:800000}]

Map<String, VSalesMonthlySummary> salesMap = salesList.stream()
    .collect(Collectors.toMap(
        VSalesMonthlySummary::getMonth,   // キー抽出: "2025-07" 等
        s -> s,                            // 値: 最初のレコードをそのまま
        (s1, s2) -> {                     // 衝突（同一月）時のマージ
            s1.setSalesTotal(s1.getSalesTotal().add(s2.getSalesTotal()));
            return s1;  // s1 を破壊的に更新して返却
        }
    ));

// 出力例:
// {"2025-07": {salesTotal:2000000}, "2025-08": {salesTotal:2800000}}
```

**注意**: マージ関数は第1引数の `salesTotal` を破壊的に更新する。元の `salesList` 内のオブジェクトが変更される副作用がある。

### 4.6 Chart.js 向け JSON構造

```javascript
            {
                label: "今期",
                backgroundColor: "rgb(255, 0, 0)",    // 赤
                lineTension: 0,
                fill: false,
                data: [2000000, 2800000, 1900000, ...]  // BigDecimal[] → 数値配列

            {

```

**Chart.jsオプション設計**:

| オプション | 値 | 説明 |

### 5.4 checkJobStatus() 関数設計

```javascript
var checkJobStatus = function (jobId, statusTextSelector) {
    $.ajax({
        type: 'GET',
        url: '/job/status/' + jobId,
        success: function (data) {
            switch (data) {
                case 'COMPLETED':
                    document.querySelector(statusTextSelector).innerText = "完了";
                    break;
                case 'FAILED':
                    document.querySelector(statusTextSelector).innerText = "実行失敗";
                    break;
                default:
                    // 再帰的にポーリング（5秒後）
                    setTimeout(function () {
                        checkJobStatus(jobId, statusTextSelector);
                    }, 5000);
                    break;
            }
        },
        error: function (jqXHR, textStatus, errorThrown) {
            document.querySelector(statusTextSelector).innerText
                = "エラー発生: " + errorThrown;
        }
    });
};
```

```javascript
    var element = $('ul.nav a').filter(function() {
        // 完全一致 OR 前方一致 でURLを照合
        return this.href == url || url.href.indexOf(this.href) == 0;
    })
    // [STEP 2] マッチした <a> に active クラスを付与
    .addClass('active')
    // [STEP 3] 親の <li> → 親の <ul> に in クラスを付与（展開状態にする）
    .parent()        // <li>
    .parent()        // <ul class="nav nav-second-level">
    .addClass('in')  // MetisMenuの展開クラス
    .parent();       // <li>（L1メニュー）

    // [STEP 4] L1メニューの <li> にも active クラスを付与
    if (element.is('li')) {
        element.addClass('active');
    }
});
```

#### 会社マスタ検索ポップアップの処理シーケンス

```javascript
$(".companySearch").click(function () {
    // [1] 現在の選択値を取得
    var companyNo = $("#companySelect").val();
    var companyType = $("#companyType").val();

    // [2] URLクエリパラメータ付きでポップアップオープン
    var url = 'companyMasterPop?companyNo=' + companyNo + '&companyType=' + companyType;
    window.open(url, 'companyMasterPop', 'width=800,height=600,scrollbars=yes');

    // [3] 同じウィンドウ名をターゲットにフォーム送信
    var form = document.forms['searchForm'];
    form.action = '/companyMasterPop';
    form.target = 'companyMasterPop';
    form.method = 'post';
    form.submit();
});
```
