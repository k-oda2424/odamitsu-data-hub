#### 1.1.3 計算メソッド: getPieceNum()

全単位の在庫を個（バラ）数に換算した合計数量を返す。`private` メソッド。

```
関数 getPieceNum(): BigDecimal
  unit1Piece = ZERO
  IF unit1StockNum != null AND goodsUnit1 != null AND goodsUnit1.containNum != null THEN
    unit1Piece = unit1StockNum * goodsUnit1.containNum
  END IF

  unit2Piece = ZERO
  IF unit2StockNum != null AND goodsUnit2 != null AND goodsUnit2.containNum != null THEN
    unit2Piece = unit2StockNum * goodsUnit2.containNum
  END IF

  unit3Piece = ZERO
  IF unit3StockNum != null AND goodsUnit3 != null AND goodsUnit3.containNum != null THEN
    unit3Piece = unit3StockNum * goodsUnit3.containNum
  END IF

  RETURN unit1Piece + unit2Piece + unit3Piece
```

**設計ポイント**:
- 各単位ごとに `stockNum`、`goodsUnit`、`containNum` の3項目すべてが非nullであることを確認
- いずれかがnullの場合、その単位からの貢献分は `BigDecimal.ZERO` として扱う
- 通常、`piece` の `containNum` は `1` であるため、`unit1Piece = unit1StockNum * 1 = unit1StockNum` となる

#### 1.1.4 計算メソッド: getEnoughStock()

在庫過不足数量を返す。在庫一覧画面の「過不足」列で使用される。`public` メソッド。

```
関数 getEnoughStock(): BigDecimal
  IF tShopAppropriateStock == null OR tShopAppropriateStock.appropriateStock == null THEN
    RETURN null
  END IF
  RETURN getPieceNum() - tShopAppropriateStock.appropriateStock
```

**設計ポイント**:
- 適正在庫が未設定の場合はnullを返す（画面側で「0」表示に変換）
- 正の値 = 在庫過多、負の値 = 在庫不足

### 5.3 登録処理フロー

```
[ブラウザ] --POST /stockCreate--> [TStockCreateController.searchGoods(form, result, model)]
  |
  +-- model("companyMap") <-- commonService.getCompanyMap()
  +-- model("warehouseMap") <-- commonService.getWarehouseMap(form.companyNo)
  +-- model("stockCreateForm") <-- form
  |
  +-- STEP 1: Spring標準バリデーション
  |   IF bindingResult.hasErrors() THEN RETURN エラー画面
  |
  +-- STEP 2: カスタムバリデーション (validate メソッド)
  |   IF form.companyNo == null THEN
  |     model("messageWarning") = "会社が選択されていません。"
  |     RETURN エラー画面
  |   IF form.goodsNo == null THEN
  |     model("messageWarning") = "商品が選択されていません。"
  |     RETURN エラー画面
  |   IF form.warehouseNo == null THEN
  |     model("messageWarning") = "倉庫が選択されていません。"
  |     RETURN エラー画面
  |
  +-- STEP 3: 重複チェック
  |   stockList = tStockService.find(form.goodsNo, form.companyNo, form.warehouseNo, null, Flag.NO)
  |   IF stockList が空でない THEN
  |     model("messageWarning") = "すでに在庫登録されています。"
  |     RETURN エラー画面
  |
  +-- STEP 4: 商品単位チェック
  |   goodsUnitList = mGoodsUnitService.findByGoodsNo(form.goodsNo)
  |   FOR EACH goodsUnit IN goodsUnitList DO
  |     IF goodsUnit.unit == "case" THEN caseUnit = goodsUnit
  |     IF goodsUnit.unit == "piece" THEN pieceUnit = goodsUnit
  |   END FOR
  |   IF caseUnit == null OR pieceUnit == null THEN
```

### 8.4 個別引当処理（stockProcess）

**トランザクション**: `@Transactional(propagation = Propagation.REQUIRES_NEW)` により注文明細ごとに独立トランザクション。

```
関数 stockProcess(tOrderDetail):
  TRY
    // STEP A: 在庫不足リストチェック
    IF shortGoodsNoList に tOrderDetail.goodsNo が含まれる THEN
      tOrderDetail.orderDetailStatus = "01" (入荷待ち)
      orderDetailService.update(tOrderDetail)
      RETURN
    END IF

    // STEP B: 手入力商品の特別処理
    IF tOrderDetail.goodsName に "手入力" が含まれる THEN
      tOrderDetail.orderDetailStatus = "10" (在庫引当)
      orderDetailService.update(tOrderDetail)
      RETURN
    END IF

    // STEP C: 在庫検索
    key = StockListKey(goodsNo, shopNo)
    IF stockListMap に key が存在する THEN
      stockList = stockListMap.get(key)
    ELSE
      stockList = moveStockList から
        goodsNo一致 AND companyNo一致 (shopMap経由) のものをフィルタ
      IF stockList が空 THEN
        stockList = StockManager.findStock(goodsNo, shopMap.get(shopNo))
      END IF
    END IF

    // STEP D: 在庫未登録チェック
    IF stockList が空 THEN
      shortGoodsNoList に goodsNo を追加
      tOrderDetail.orderDetailStatus = "01" (入荷待ち)
      ログ出力: "在庫が登録されていません。..."
      orderDetailService.update(tOrderDetail)
      RETURN
    END IF

    // STEP E: 引当実行 (マイナス在庫を許容)
    targetStock = stockList.get(0)  // 先頭の倉庫を使用
    orderNum = tOrderDetail.orderNum - cancelNum - returnNum
    moveNum = -1 * orderNum  // 減算のため負の値

    updateStock = StockManager.move(
      companyNo: targetStock.companyNo,
      goodsNo: tOrderDetail.goodsNo,
      warehouseNo: targetStock.warehouseNo,
      moveTime: tOrderDetail.orderDateTime,
      reason: StockLogReason.ALLOCATE,
      moveNum: moveNum,
      deliveryNo: tOrderDetail.deliveryNo,
      destinationWarehouseNo: null,
      purchaseNo: null,
      overWriteFlg: false
    )

    IF updateStock == null THEN
      stockListMap.put(key, stockList)
      RETURN  // 移動数量が0の場合
    END IF

    // STEP F: ステータス更新
    stockList.set(0, updateStock)  // キャッシュ更新
    tOrderDetail.orderDetailStatus = "10" (在庫引当)
    orderDetailService.update(tOrderDetail)

  CATCH Exception e
    ログ出力: "在庫引当処理に失敗しました。..."
  END TRY
```

### 8.5 3単位在庫モデルにおける引当計算

引当時の在庫減算は `StockManager.move()` 内で以下のように処理される:

1. 注文数量 `orderNum` はバラ数として扱われる
2. `StockManager.calculateMinGoodsUnitNum()` で現在庫をバラ数に換算
3. 現在庫(バラ) + 移動数(負のバラ) = 新在庫(バラ)
4. `calculateGoodsUnit()` で新在庫(バラ)をケース/バラに再分配

```
// バラからケース/バラへの再分配
unit2Division = newPieceNum.divideAndRemainder(caseUnit.containNum)
unit2StockNum = unit2Division[0]  // ケース数
unit1StockNum = unit2Division[1]  // バラ余り

// unit3が存在する場合はさらに分割
IF unit3 存在 THEN
  unit3Division = unit2Num.divideAndRemainder(unit3.containNum)
  unit3StockNum = unit3Division[0]
  unit2StockNum = unit3Division[1]
END IF
```

### 10.3 ShopAppropriateStockCalculateTasklet 計算アルゴリズム

#### ジョブパラメータ

| パラメータ名 | 型 | 説明 | 注入方法 |
|---|---|---|---|
| `spanMonths` | `Integer` | 過去何ヶ月分のデータで計算するか | `@Value("#{jobParameters['spanMonths']}")` |

#### 計算フロー疑似コード

```
関数 execute(contribution, chunkContext):
  // STEP 1: TRUNCATE
  truncateAppropriateStock()
    -> TShopAppropriateStockService.truncateTShopAppropriateStock()

  // STEP 2: 対象データ取得
  targetMap = getTargetMap()

  // STEP 3: 期間日数算出
  periodDays = ChronoUnit.DAYS.between(
    LocalDate.now().minusMonths(spanMonths),
    LocalDate.now()
  )

  // STEP 4: 商品ごとに計算
  FOR EACH (goodsNo, dataMap) IN targetMap DO
    targetDataList = dataMap.values().stream().map(Optional::get).collect(toList())
    leadTime = targetDataList[0].leadTime

    // 計算オブジェクト生成
    calc = new AppropriateStockCalculate(leadTime)

    // 4a: 標準偏差計算
    standardDeviation = calc.calculateStandardDeviation(targetDataList, periodDays)

    // 4b: 安全在庫計算
    safetyStock = calc.calculateSafetyStock(standardDeviation)

    // 4c: 1日平均出荷数量計算
    totalShipment = SUM(targetDataList.goodsNum)
    ave = totalShipment / periodDays  // 小数点以下32桁精度

    // 4d: 適正在庫計算
    appropriateStock = calc.calculateAppropriateStock(ave, safetyStock)

    // 4e: 登録 (小数点以下3桁, 四捨五入)
    updateAppropriateStock(goodsNo, shopNo, appropriateStock, safetyStock)
  END FOR
```

#### getTargetMap() の詳細

```
関数 getTargetMap():
  // 1. 対象注文明細の抽出
  orderDateFrom = LocalDateTime.now().truncatedTo(DAYS).minusMonths(spanMonths)
  orderDetailList = TOrderDetailService.find(
    orderDetailStatus: ["10", "20"],  // 在庫引当 + 納品済
    orderDateTimeFrom: orderDateFrom,
    orderDateTimeTo: now,
    delFlg: Flag.NO
  )

  // 2. 直送フラグ除外
  targetList = orderDetailList
    .filter(od -> od.tDelivery.directShippingFlg != "1")

  // 3. リードタイムの一括検索 (パフォーマンス最適化)
  goodsNoList = targetList.map(TOrderDetail::getGoodsNo)
  IF goodsNoList.size() > 1000 THEN
    1000件ずつに分割して setLeadTimeMap() を呼び出す
  ELSE
    setLeadTimeMap(goodsNoList)
  END IF

  // 4. 商品ごと・日付ごとに集計
  RETURN targetList
    .map(od -> AppropriateStockEntity(
      goodsNum: od.orderNum - od.cancelNum - od.returnNum,
      goodsNo: od.goodsNo,
      moveDate: od.orderDate,
      leadTime: getLeadTime(od.goodsNo, od.shopNo),
      shopNo: od.shopNo
    ))
    .groupBy(goodsNo)
    .thenGroupBy(moveDate)
    .reducing(同一日付の数量を加算)
```

### 10.4 標準偏差計算 (calculateStandardDeviation)

```
関数 calculateStandardDeviation(targetList, periodDays):
  // 日次出荷数量リストを作成
  dataList = targetList.map(entity -> entity.goodsNum)

  // 出荷実績のない日をゼロ補完
  targetDataNum = dataList.size()
  FOR i = 1 TO (periodDays - targetDataNum) DO
    dataList.add(BigDecimal.ZERO)
  END FOR

  // BigDecimal -> Double[] に変換
  dataArray = dataList.map(BigDecimal::doubleValue).toArray()

  // StaticsCalculator で標準偏差を計算
  stdDev = StaticsCalculator.getStandardDeviation(dataArray)

  RETURN BigDecimal.valueOf(stdDev)
```

### 10.5 安全在庫計算 (calculateSafetyStock)

```
定数 SERVICE95 = 1.645  // 95%サービスレベルに対応する安全係数

関数 calculateSafetyStock(standardDeviation):
  sqrtLeadTime = Math.sqrt(leadTime)
  safetyStock = SERVICE95 * standardDeviation * sqrtLeadTime
  RETURN safetyStock を 小数点以下2桁で四捨五入
```

### 10.6 適正在庫計算 (calculateAppropriateStock)

```
関数 calculateAppropriateStock(averageDailyShipment, safetyStock):
  leadTimeStock = averageDailyShipment * leadTime
  appropriateStock = leadTimeStock + safetyStock
  RETURN appropriateStock を 小数点以下2桁で四捨五入
```

### 10.7 リードタイム取得ロジック (getLeadTime)

```
関数 getLeadTime(goodsNo, shopNo):
  key = MSalesGoodsPK(goodsNo, shopNo)

  // キャッシュチェック
  IF leadTimeMap に key が存在する THEN
    RETURN leadTimeMap.get(key) ?? 7  // nullの場合はデフォルト7

  // 販売商品マスタから検索
  mSalesGoods = MSalesGoodsService.getByPK(shopNo, goodsNo)
  IF mSalesGoods != null AND mSalesGoods.leadTime != null THEN
    leadTimeMap.put(key, mSalesGoods.leadTime)
    RETURN mSalesGoods.leadTime

  // 販売商品ワークから検索
  wSalesGoods = WSalesGoodsService.getByPK(shopNo, goodsNo)
  IF wSalesGoods != null AND wSalesGoods.leadTime != null THEN
    leadTimeMap.put(key, wSalesGoods.leadTime)
    RETURN wSalesGoods.leadTime

  // 見つからない場合はデフォルト値
  RETURN 7  // 1週間
```

## 11. 3単位在庫モデル設計

### 11.1 概念設計

在庫は最大3種類の単位で管理される。各単位は `m_goods_unit` テーブルで商品ごとに定義される。

```
+-------------------------------------------+
|             3単位在庫モデル                 |
+-------------------------------------------+
| unit3 (第3単位)  | 将来拡張用。現在非表示    |
|   unit3_no       | unit3_stock_num           |
+------------------+---------------------------+
| unit2 (ケース)   | 1ケース = N個             |
|   unit2_no       | unit2_stock_num           |
+------------------+---------------------------+
| unit1 (バラ/個)  | 最小単位。containNum = 1   |
|   unit1_no       | unit1_stock_num           |
+------------------+---------------------------+
```

### 11.2 UnitType 列挙型

| 定数 | `getValue()` | 説明 |
|---|---|---|
| `CASE` | `"case"` | ケース単位 |
| `PIECE` | `"piece"` | バラ/個数単位 |

### 11.3 MGoodsUnit（商品単位マスタ）

| カラム名 | 型 | 説明 |
|---|---|---|
| `unit_no` | Integer (PK) | 単位番号 (シーケンス) |
| `goods_no` | Integer | 商品番号 |
| `unit` | String | 単位種別 ('case' / 'piece') |
| `contain_num` | BigDecimal | 入数 (例: 1ケース=30個なら30、バラの場合は1) |
| `parent_unit_no` | Integer | 親単位番号 (ケースの parent が存在する場合は第3単位) |

### 11.4 単位間の親子関係

```
piece (バラ) -- parentUnitNo --> case (ケース) -- parentUnitNo --> 第3単位
  containNum = 1                 containNum = N                  containNum = M

例: 商品「パッドウルトラ」
  piece: unit_no=101, containNum=1, parentUnitNo=102
  case:  unit_no=102, containNum=30, parentUnitNo=null
  => 1ケース = 30個
```

### 11.5 個換算ロジック (TStock.getPieceNum)

```
個換算在庫 = unit1StockNum * unit1.containNum  (通常: N * 1 = N個)
           + unit2StockNum * unit2.containNum  (通常: M * 30 = M*30個)
           + unit3StockNum * unit3.containNum  (通常: 0)
```

### 11.6 バラからケース/バラへの再分配ロジック (StockManager.calculateGoodsUnit)

```
関数 calculateGoodsUnit(iStockEntity, goodsUnitList, moveNum):
  // 最小単位 (piece) を特定
  minGoodsUnit = goodsUnitList
    .filter(unit -> unit.unit == "piece")
    .findFirst()
  iStockEntity.unit1No = minGoodsUnit.unitNo

  // ケース単位を特定
  goodsUnit2 = goodsUnitList
    .filter(unit -> unit.unitNo == minGoodsUnit.parentUnitNo)
    .filter(unit -> unit.containNum > 1)  // containNum=1は実質バラなので除外
    .findFirst()

  IF goodsUnit2 == null THEN
    // ケース単位なし: バラのみで登録
    iStockEntity.unit1StockNum = moveNum
    RETURN iStockEntity
  END IF

  iStockEntity.unit2No = goodsUnit2.unitNo

  // 第3単位を特定
  goodsUnit3 = goodsUnitList
    .filter(unit -> unit.unitNo == goodsUnit2.parentUnitNo)
    .findFirst()

  // ケースとバラに分割
  [unit2Num, unit1Num] = moveNum.divideAndRemainder(goodsUnit2.containNum)

  IF goodsUnit3 == null THEN
    iStockEntity.unit1StockNum = unit1Num
    iStockEntity.unit2StockNum = unit2Num
    RETURN iStockEntity
  END IF

  // 第3単位とケースに分割
  iStockEntity.unit3No = goodsUnit3.unitNo
  [unit3Num, unit2Num_rest] = unit2Num.divideAndRemainder(goodsUnit3.containNum)

  iStockEntity.unit1StockNum = unit1Num
  iStockEntity.unit2StockNum = unit2Num_rest
  iStockEntity.unit3StockNum = unit3Num
  RETURN iStockEntity
```

### 11.7 バラ数への逆換算ロジック (StockManager.calculateMinGoodsUnitNum)

```
関数 calculateMinGoodsUnitNum(stock):
  // 商品単位の入数マップを取得/キャッシュ
  unitNoList = [stock.unit1No, stock.unit2No, stock.unit3No] (nullを除く)

  unit2ContainNum = ZERO
  unit3ContainNum = ZERO

  // unitNoMapキャッシュまたはDB検索でcontainNumを取得
  FOR EACH unitNo IN unitNoList DO
    IF unitNo == stock.unit2No THEN unit2ContainNum = containNum
    IF unitNo == stock.unit3No THEN unit3ContainNum = containNum
  END FOR

  // バラ数計算
  stockNum = stock.unit1StockNum
  IF stock.unit2StockNum != null THEN
    stockNum += unit2ContainNum * stock.unit2StockNum
  IF stock.unit3StockNum != null THEN
    stockNum += unit3ContainNum * stock.unit3StockNum

  RETURN stockNum
```
