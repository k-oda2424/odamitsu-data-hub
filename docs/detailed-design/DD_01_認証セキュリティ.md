### 1.3 configure(WebSecurity) メソッド処理

```
メソッド: configure(WebSecurity web)
目的: 静的リソースをSpring Securityのフィルターチェーンから完全に除外する

処理:
  web.ignoring().antMatchers(
    "/favicon.ico",
    "/css/**",
    "/js/**",
    "/images/**",
    "/fonts/**"
  )

効果:
  - 上記パスへのリクエストはSecurityFilterChainを一切通過しない
  - 認証チェック不要、CSRFトークン検証不要
  - パフォーマンス向上（フィルター処理のオーバーヘッド排除）
```

### 1.4 configure(HttpSecurity) メソッド処理

```
メソッド: configure(HttpSecurity http)
目的: HTTPリクエストに対するセキュリティルールの定義

処理（疑似コード）:

  // === 1. URL認可設定 ===
  http.authorizeRequests()
      .antMatchers("/loginForm").permitAll()       // ログインフォームは未認証でもアクセス可
      // .antMatchers("/admin/**").permitAll()      // [コメントアウト] 将来の管理者URL
      .anyRequest().authenticated()                 // その他全URLは認証必須

  // === 2. フォームログイン設定 ===
  http.formLogin()
      .loginProcessingUrl("/login")                 // POST /login で認証処理を実行
      .loginPage("/loginForm")                      // 未認証時にリダイレクトするログインページURL
      .failureUrl("/loginForm?error")               // 認証失敗時のリダイレクト先
      .successHandler(customAuthenticationSuccessHandler)  // 認証成功時のカスタムハンドラー
      .defaultSuccessUrl("/dashboard", true)         // デフォルトの成功リダイレクト先
      .usernameParameter("login_id")                 // HTMLフォームのユーザー名パラメータ名
      .passwordParameter("password")                 // HTMLフォームのパスワードパラメータ名

  // === 3. ログアウト設定 ===
  http.logout()
      .logoutRequestMatcher(new AntPathRequestMatcher("/logout"))  // POST/GET両方で /logout を受付
      .logoutSuccessUrl("/loginForm")               // ログアウト成功後の遷移先
      .deleteCookies("JSESSIONID")                  // Cookieの削除
      .invalidateHttpSession(true)                  // セッション破棄
      .permitAll()                                  // ログアウト処理自体は認証不要

  // === 4. セッションタイムアウトフィルター追加 ===
  http.addFilterBefore(
      new SessionTimeoutFilter(),                   // インスタンスを直接生成（Spring管理外）
      UsernamePasswordAuthenticationFilter.class     // 認証フィルターの直前に挿入
  )

  // === 5. セッション管理設定 ===
  http.sessionManagement()
      .maximumSessions(1)                           // 同一ユーザーの同時セッション数: 1
      .maxSessionsPreventsLogin(false)              // 新ログイン許可（旧セッション無効化）
      .and()
      .sessionCreationPolicy(IF_REQUIRED)           // セッション作成ポリシー: 必要時のみ
      .invalidSessionStrategy(sessionTimeoutRedirectStrategy)  // セッション無効時の処理戦略
      .sessionFixation().changeSessionId()          // セッション固定攻撃対策: セッションID変更
```

### 3.3 loadUserByUsername() 処理フロー

```
メソッド: loadUserByUsername(String loginId)
戻り値: UserDetails
例外: UsernameNotFoundException

入力: loginId - ログインフォームから入力されたログインID

処理フロー:
  1. MLoginUser user = null を初期化

  2. try:
       user = loginUserRepository.findByLoginId(loginId)
       // SQL: SELECT * FROM m_login_user WHERE login_id = :loginId
     catch (Exception e):
       throw new UsernameNotFoundException("It can not be acquired User")
       // DB接続エラーやその他のシステム例外を認証失敗に変換

  3. if (user == null):
       throw new UsernameNotFoundException("User not found for login id: " + loginId)
       // 該当ユーザーが存在しない場合

  4. return new LoginUser(user)
       // LoginUser コンストラクタ内でROLE_USERを付与
       // super(user.getLoginId(), user.getPassword(),
       //       AuthorityUtils.createAuthorityList("ROLE_USER"))
```

**DBクエリ詳細**:

```sql
-- LoginUserRepository#findByLoginId() のSpring Data JPA自動生成クエリ
SELECT m.*
FROM m_login_user m
WHERE m.login_id = ?1
```

**注意事項**:
- `findByLoginId` は `MLoginUser` を単一で返す（`List`ではない）
- `login_id` カラムにユニーク制約がDBレベルで設定されている前提
- `del_flg` による論理削除チェックは `loadUserByUsername` 内では行っていない（削除済みユーザーでもログイン可能な点に注意）

### 3.5 find() 処理フロー

```
メソッド: find(Integer loginUserNo, String userName, String loginId, Flag delFlg)
戻り値: List<MLoginUser>

処理:

    WHERE句 =
      loginUserNo が非nullなら AND login_user_no = :loginUserNo

```

### 3.6 insert() 処理フロー

```
メソッド: insert(LoginUserCreateForm loginUserCreateForm)
戻り値: MLoginUser
例外: Exception

処理:
  1. MLoginUser saveLoginUser = new MLoginUser()
  2. BeanUtils.copyProperties(loginUserCreateForm, saveLoginUser)
     // loginId, password, userName, priority がコピーされる
  3. encodedPassword = passwordEncoder.encode(saveLoginUser.getPassword())
     // BCryptハッシュ化
  4. saveLoginUser.setPassword(encodedPassword)
  5. saveLoginUser.setCompanyNo(1)          // 固定値: 会社番号1
  6. saveLoginUser.setCompanyType("admin")  // CompanyType.ADMIN.getValue()
  7. return this.insert(this.loginUserRepository, saveLoginUser)
     // CustomService#insert() を呼出
     //   → addUserNo にログインユーザー番号を設定
     //   → addDateTime に現在日時を設定
     //   → delFlg が null なら "0" を設定
     //   → repository.save(entity) で永続化
```

### 3.7 update() 処理フロー

```
メソッド: update(LoginUserModifyForm loginUserModifyForm)
戻り値: MLoginUser
例外: Exception

処理:
  1. saveLoginUser = loginUserRepository.findByLoginId(loginUserModifyForm.getLoginId())
     // 既存ユーザーをDBから取得

  2. if (loginUserModifyForm.getPassword() != null):
       encodedPassword = passwordEncoder.encode(loginUserModifyForm.getPassword())
       saveLoginUser.setPassword(encodedPassword)
     // パスワードがnullの場合は更新しない（既存パスワードを維持）

  3. return this.update(this.loginUserRepository, saveLoginUser)
     // CustomService#update() を呼出
     //   → modifyUserNo にログインユーザー番号を設定
     //   → modifyDateTime に現在日時を設定
     //   → repository.save(entity) で永続化

注意: userName や loginId の更新は行われない
      （findByLoginId で取得した既存エンティティのフィールドはそのまま維持）
```

## 4. SessionTimeoutFilter 処理設計

### 4.1 クラス基本情報

| 項目 | 値 |
|---|---|
| パッケージ | `jp.co.oda32.filter` |
| クラス名 | `SessionTimeoutFilter` |
| 継承 | `OncePerRequestFilter` |
| Spring管理 | **非管理**（`new SessionTimeoutFilter()` で直接生成） |
| フィルター位置 | `UsernamePasswordAuthenticationFilter` の直前 |
| ソースファイル | `src/main/java/jp/co/oda32/filter/SessionTimeoutFilter.java` |

### 4.2 EXCLUDED_URLS 定義

```java
private static final List<String> EXCLUDED_URLS = Arrays.asList(
    "/loginForm", "/login", "/logout", "/error", "/dashboard",
    "/css/", "/js/", "/images/", "/fonts/", "/favicon.ico",
    "/api/", "/batch"
);
```

**判定方法**: `url.startsWith(excludedUrl)` — 前方一致マッチ

| 除外URL | マッチ例 |
|---|---|
| `/loginForm` | `/loginForm`, `/loginForm?error` |
| `/login` | `/login`, `/loginUserList` (**注意**: `/login` で始まる全URL) |
| `/logout` | `/logout` |
| `/error` | `/error`, `/error/404` |
| `/dashboard` | `/dashboard`, `/dashboard/summary` |
| `/css/` | `/css/style.css` |
| `/js/` | `/js/app.js` |
| `/images/` | `/images/logo.png` |
| `/fonts/` | `/fonts/arial.woff` |
| `/favicon.ico` | `/favicon.ico` |
| `/api/` | `/api/v1/data` |
| `/batch` | `/batch/run` |

**重要な注意点**: `/login` が除外URLに含まれるため、`/loginUserList`、`/loginUserCreateForm`、`/loginUserModifyForm` 等もURL記録対象外となる。

### 4.3 doFilterInternal() 完全分岐フロー図

```
doFilterInternal(request, response, filterChain)
    |
    v
[1] requestURI = request.getRequestURI()
    contextPath = request.getContextPath()
    path = requestURI.substring(contextPath.length())
    |
    v
[2] authentication = SecurityContextHolder.getContext().getAuthentication()
    isAuthenticated = (authentication != null
                       && authentication.isAuthenticated()
                       && !"anonymousUser".equals(authentication.getPrincipal().toString()))
    |
    +---[YES: isAuthenticated == true]---+
    |                                     |
    |                                     v
    |                              [3] isExcludedUrl(path)?
    |                                     |
```

### 8.1 BCrypt処理フロー

#### 8.1.1 パスワード登録時のハッシュ化フロー

```
[ユーザー入力: "mypassword123"]
    |
    v
[LoginUserService#insert()]
    |
    v
[passwordEncoder.encode("mypassword123")]
    |
    v
[BCryptPasswordEncoder 内部処理]
    |
    +-- 1. ランダムソルト生成（16バイト）
    |      例: "$2a$10$N9qo8uLOickgx2ZMRZoMye"
    |
    +-- 2. strength = 10（デフォルト）
    |      ハッシュ反復回数 = 2^10 = 1024回
    |
    +-- 3. BCryptハッシュ計算
    |      input: "mypassword123" + ソルト
    |
    +-- 4. ハッシュ文字列生成
    |      形式: "$2a$10$<ソルト22文字><ハッシュ31文字>"
    |      例: "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy"
    |      全長: 60文字
    |
    v
[saveLoginUser.setPassword(ハッシュ文字列)]
    |
    v
[DB保存: m_login_user.password カラム]
```

#### 8.1.2 ログイン時のパスワード照合フロー

```
[ユーザー入力: "mypassword123"]
    |
    v
[Spring Security 内部: DaoAuthenticationProvider]
    |
    v
[passwordEncoder.matches("mypassword123", DBから取得したハッシュ)]
    |
    v
[BCryptPasswordEncoder#matches() 内部処理]
    |
    +-- 1. DBハッシュからソルト部分を抽出
    |      "$2a$10$N9qo8uLOickgx2ZMRZoMye"
    |
    +-- 2. 入力パスワードを抽出したソルトでハッシュ化
    |      BCrypt.hashpw("mypassword123", "$2a$10$N9qo8uLOickgx2ZMRZoMye")
    |
    +-- 3. 生成されたハッシュとDBハッシュを比較
    |      一致 → return true（認証成功）
    |      不一致 → return false（認証失敗）
    |
    v
[認証結果]
```

### 8.2 OldPasswordValidator 実装詳細

#### 8.2.1 クラス基本情報

| 項目 | 値 |
|---|---|

| ソースファイル | `src/main/java/jp/co/oda32/domain/validation/OldPasswordValidator.java` |

#### 8.2.2 isValid() 処理フロー

```
isValid(Object value, ConstraintValidatorContext context)
    |
    v
[1] BeanWrapper beanWrapper = new BeanWrapperImpl(value)
    // value = LoginUserModifyForm オブジェクト
    |
    v
[2] fieldValue = beanWrapper.getPropertyValue("oldPassword")
    // ユーザーが入力した旧パスワード（平文）
    |
    v
[3] try:
      loginUser = loginUserService.getLoginUser()
      // SecurityContextHolder からログイン中ユーザーの MLoginUser を取得
    catch (Exception e):
      this.message = e.getMessage()
      return false
    |
    v
[4] encodedPassword = passwordEncoder.encode(loginUser.getPassword())
    // ★ 問題点: DBに保存されたハッシュ値をさらにencode()している
    // loginUser.getPassword() = "$2a$10$xxxx..." (既にハッシュ済み)
    // encodedPassword = "$2a$10$yyyy..." (ハッシュのハッシュ)
    |
    v
[5] encodedInputPassword = passwordEncoder.encode((String) fieldValue)
    // ユーザー入力の旧パスワード（平文）をencode()
    // encodedInputPassword = "$2a$10$zzzz..."
    |
    v
[6] Objects.equals(encodedPassword, encodedInputPassword)?
    // ★ 問題点: BCryptは同一平文でも毎回異なるソルトで異なるハッシュを生成する
    // そのため、この比較は常に false になる
    |
    +---[true]--→ return true（実質到達不可能）
    |
    +---[false]
         |
         v
    context.disableDefaultConstraintViolation()
    context.buildConstraintViolationWithTemplate(message)
           .addPropertyNode("oldPassword")
           .addConstraintViolation()
    return false
```

#### 8.2.3 問題点の詳細と修正案

**現在のコード（不正）**:

```java
// 問題のあるコード
String encodedPassword = passwordEncoder.encode(loginUser.getPassword());
String encodedInputPassword = passwordEncoder.encode((String) fieldValue);
if (Objects.equals(encodedPassword, encodedInputPassword)) {
    return true;
}
```

**問題点**:
1. `loginUser.getPassword()` は既にBCryptハッシュ済みの文字列（例: `$2a$10$...`）
2. これを再度 `encode()` すると、ハッシュのハッシュが生成される
3. `encode()` はランダムソルトを使用するため、同一入力でも毎回異なる出力となる
4. `Objects.equals()` による文字列比較は常に `false` となる
5. **結果として、旧パスワードの検証は常に失敗する**

**正しい実装**:

```java
// 修正案
String storedHashedPassword = loginUser.getPassword();
if (passwordEncoder.matches((String) fieldValue, storedHashedPassword)) {
    return true;
}
```

`matches(rawPassword, encodedPassword)` メソッドを使用することで:
1. `encodedPassword` からソルトを抽出
2. `rawPassword` を抽出したソルトでハッシュ化
3. 生成されたハッシュと `encodedPassword` を比較
