### 真の原因 (仮説検証済み)

これらは振込明細 Excel (2026-03-05, 2026-03-19 発行) 作成時点ではSMILEに未登録だった正当な新規スリップで、
ext_purchase_no の重複もない (= `t_purchase_ext_purchase_no_shop_no_uindex` UNIQUE 制約 OK)。

## 7. 推奨アクション

### 7.1 DIFFの本質と選択肢

**DB集計値は「間違い」ではない**。SMILE側で遅れて追加取込された正当な新規スリップがDBに含まれており、
振込明細Excel（発行2026-03-05/03-19）と物理的にズレている。以下どちらの運用にするか意思決定が必要:

**(A) Excel 振込実績を正とする（支払実行ベース）**

```sql
-- UI: /finance/accounts-payable で各DIFF行の tax_included_amount_change を Excel 値に手動調整
-- verified_manually は既に true なので Calculator は上書きしない
-- April late-import (税抜 6,541,012 / 税込≈ 7,195,113) は tx_month=2026-03-20 に繰越
```

**(B) SMILE実績(DB)を正とし、DIFF を強制MATCH化**

```sql
UPDATE t_accounts_payable_summary
SET verification_result=1, mf_export_enabled=true,
    verification_note='2026-04-14/15 SMILE追加取込分を含む'
WHERE shop_no=1 AND transaction_month='2026-02-20'
  AND verification_result=0 AND verified_manually=true;
```

### 7.2 親支払先 NULL の m_supplier 修正 (小額だが集計漏れ)

```sql
UPDATE m_supplier SET payment_supplier_no=67, modify_date_time=NOW(), modify_user_no=1
  WHERE shop_no=1 AND supplier_no=351; -- 三興化学工業 +81,735
UPDATE m_supplier SET payment_supplier_no=50, modify_date_time=NOW(), modify_user_no=1
  WHERE shop_no=1 AND supplier_no=341; -- 日本製紙クレシア +58,362
UPDATE m_supplier SET payment_supplier_no=120, modify_date_time=NOW(), modify_user_no=1
  WHERE shop_no=1 AND supplier_no=392; -- 西日本衛材 +23,500
UPDATE m_supplier SET payment_supplier_no=6, modify_date_time=NOW(), modify_user_no=1
  WHERE shop_no=1 AND supplier_no=308; -- イトマン +12,650
-- 適用後 2026-02-20 集計を再実行すると追加取込される (ただし (A) を選ぶならやらない方が良い)
```

### 7.3 恒久対策（中期）

- **遅延取込の検知**: `t_purchase.add_date_time > 翌月20日` の件数を tx_month 単位で監視しUIで警告表示
- **m_supplier.payment_supplier_no NULL アラート**: 現在 114/280 が NULL。新規仕入で NULL 親の
  supplier が使われたら `AccountsPayableSummaryCalculator` が警告ログを出す(既存)→ 定期監視に昇格
- **SmilePaymentVerifier は空回り**: `t_smile_payment` は 2025-06 以降データなし。UI のSMILE検証ボタンを非表示化、
  Excel 起点の `PaymentMfImportService.applyVerification` に一本化
- **19日払いExcelの名前解決**: 「仕入コード」列なしのファイルは名前一致がブレやすい。
  `m_payment_mf_rule.source_name` に Excel 表記ゆれを登録推奨 (ケイティケイ/中川製袋化工/カミイソ/ハラプレックス/アトムワークス 等)
